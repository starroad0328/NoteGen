 1. 프로젝트 개요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ 서비스 목적 ]

  학생이 수업 중 작성한 불완전한 손필기를 사진으로 업로드
           ↓
  AI가 학년 수준에 맞게 정리 + 중요 포인트 자동 강조
           ↓
  하루 공부 목표 시간에 맞춰 문제 자동 제공


[ MVP 목표 ]

  ✓ 사진 업로드 → AI 정리 → 노트 보관 → 일일 학습
    완결된 학습 루프 제공

  ✓ 듀오링고식 시간 목표 기반 분량 제어 UX

  ✓ 고정 프리셋 기반으로 단순하고 예측 가능한 경험


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 2. 핵심 컨셉 요약
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  💬 "하루에 몇 분 공부할지 정하면,
      그 시간 안에 끝낼 수 있는 만큼의 문제만 나온다."

  핵심 원리:
    • 공부량은 시간 기준
    • 앱이 시간을 문제 수로 자동 변환
    • 매일 '완료 경험'을 제공

[ 4.1 목표 시간별 문제 수 ]

  ┌────────────┬────────────────┐
  │ 목표 시간   │ 제공 문제 수    │
  ├────────────┼────────────────┤
  │   1분      │     2문제      │
  │   3분      │     4문제      │
  │   5분      │     6문제      │
  │  10분      │    10문제      │
  │  15분      │    14문제      │
  │  20분      │    18문제      │
  └────────────┴────────────────┘

  특징:
    ✓ 개인별 속도 보정 없음
    ✓ 항상 동일한 분량 제공 (예측 가능한 UX)


[ 4.2 문제 타입 구성 비율 ]

  MVP 전략: 객관식 + 단답형 2가지로 단순화
           (풀이 속도와 채점 안정성 우선)

  ┌────────────┬──────────┬──────────┬──────────┐
  │ 목표 시간   │ 총 문제수 │ 객관식   │  단답형  │
  ├────────────┼──────────┼──────────┼──────────┤
  │   1분      │    2     │    1     │    1     │
  │   3분      │    4     │    2     │    2     │
  │   5분      │    6     │    3     │    3     │
  │  10분      │   10     │    6     │    4     │
  │  15분      │   14     │    8     │    6     │
  │  20분      │   18     │   10     │    8     │
  └────────────┴──────────┴──────────┴──────────┘

  문제 타입별 역할:
    • 객관식 - 빠른 판단, 리듬 형성, 부담 최소화
    • 단답형 - 핵심 개념 직접 회상

│  • 설정 화면에서 변경 가능                         │
│                                                   │
└───────────────────────────────────────────────────┘

┌─ F6. 일일 학습 (Duolingo식) ──────────────────────┐
