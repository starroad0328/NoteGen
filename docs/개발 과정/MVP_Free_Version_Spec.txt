본 문서는 자동필기앱의 **최소 기능 제품(MVP)** 범위를 정의합니다.

본 MVP는 **유료 기능은 추후 제공**함을 전제로 하며,
현재 단계에서는 **필기 자동 정리 경험 검증**에만 집중합니다.


[ 핵심 원칙 ]

  ✓ 무료 기능만 포함
  ✓ 필기 정리 경험 검증
  ✓ 빠른 시장 출시

[ Primary 타겟 ]

  👨‍🎓 중·고등학생
     → 수업 중 필기를 했으나 정리가 불완전한 사용자
     → 빠르게 정리된 노트를 확인하고 싶은 사용자


[ 사용 시나리오 ]

  1. 수업 후 칠판 사진을 찍어둠
  2. 집에 와서 앱에 업로드
  3. 정리 방식 선택 (기본 요약 or 코넬식)
  4. AI가 자동 정리
  5. 깔끔한 노트 확인

┌─────────────────────────────────────────────────────────┐
│  6.2 OCR 및 필기 인식                                   │
└─────────────────────────────────────────────────────────┘

[ 기본 처리 ]

  1. 손글씨 OCR 기반 텍스트 추출
     → 한글, 영문, 숫자 지원

  2. 인식 오류 발생 시 문맥 기반 AI 보정 수행
     → LLM으로 자연스럽게 문장 복원


[ 수식 및 기호 ]

  ⚠️  제한적 지원

  • 인식은 시도하되 정확도 보장은 하지 않음
  • 실패 시 일반 텍스트로 처리
  • 복잡한 수식은 인식 제외

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

본 MVP는 다음을 **검증하기 위한 제품**입니다.


[ 검증 목표 ]

  ❓ AI가 손필기를 실제로 쓸만한 노트로 정리해줄 수 있는가?

  ❓ 사용자가 정리 노동이 줄었다고 느끼는가?

  ❓ 반복 사용 가능성이 존재하는가?


[ 명확히 할 것 ]

  ⚠️  본 MVP는 **완성형 학습 서비스가 아닙니다.**

  ✅ 핵심 가치 검증용 최소 제품
  ✅ 사용자 반응 확인
  ✅ 빠른 개선 반복


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 11. MVP 개발 일정 (4주 권장)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ 1주차: 기본 파이프라인 ]

  • 이미지 업로드 기능
  • OCR 연동 및 테스트
  • 기본 요약 정리 1가지만 구현


[ 2주차: 정리 기능 완성 ]

  • 코넬식 정리 추가
  • AI 정리 품질 개선
  • 중요도 표시 로직


[ 3주차: 노트 뷰어 & 저장 ]

  • 읽기 전용 노트 UI
  • 자동 저장 기능
  • 노트 리스트 화면


[ 4주차: 테스트 & 개선 ]

  • 실제 필기로 품질 테스트
  • UI/UX 개선
  • 버그 수정
  • 베타 테스트


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 12. 출시 후 피드백 수집 계획
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ 수집 방법 ]

  1. 인앱 설문
     → 정리 결과 확인 후 만족도 조사

  2. 사용 로그 분석
     → 업로드 횟수, 완료율, 재방문율

  3. 사용자 인터뷰 (선택적)
     → 5~10명 심층 인터뷰


[ 주요 질문 ]

  • 정리 결과가 만족스러웠나요?
  • 다시 사용하고 싶으신가요?
  • 어떤 기능이 추가되면 좋을까요?
  • 유료로 전환 시 지불 의향이 있나요?


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 13. V2 전환 기준
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

다음 조건 충족 시 V2(유료 기능) 개발 시작:

  ✅ 누적 사용자 500명 이상
  ✅ 재사용률 60% 이상
  ✅ 유료 전환 의향 30% 이상 (설문)
  ✅ 정리 품질 만족도 75% 이상

 14. 위험 요소 및 대응
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ 위험 1: OCR 정확도 ]

  문제: 손글씨 인식 실패 시 사용자 이탈
  대응:
    • 문맥 기반 AI 보정
    • 인식 실패 시 안내 문구
    • 재촬영 유도


[ 위험 2: AI 정리 품질 ]

  문제: 정리 결과가 이상하거나 부정확
  대응:
    • 프롬프트 지속 개선
    • 사용자 피드백 반영
    • 수동 검수 (초기)


[ 위험 3: 비용 ]

  문제: OCR + AI API 비용
  대응:
    • 초기는 무료 제공 (투자)
    • V2 유료 전환으로 회수
    • 효율적인 프롬프트로 비용 절감
